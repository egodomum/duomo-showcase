## 0. 한 줄 요약

> DUOMO 상세페이지는 "매거진"이 아니라 **"e커머스 디테일 페이지"**다.
> **순백 배경 · 전부 중앙 정렬 · 큰 글씨 · 브랜드 컬러 액센트 · 거대한 여백.**

우리가 초기에 만든 premium-editorial(좌측정렬·오프화이트·골드·1200px·17px 본문)은
**잡지 톤**이라 DUOMO 실제와 다르다. 카페24/29CM/SSG에 올리는 1000px 폭 상세 이미지엔
아래 실측 스펙을 따라야 한다.

## 3. 타이포그래피 (Figma 스타일 그대로)

| 스타일명 | 폰트 | 크기(1000px 기준) | 행간 | 자간 | 용도 |
|---|---|---|---|---|---|
| Headline/H1 | **Noto Serif Display SemiBold** | 106px | 1.0 | 0 | 제품명 (예: "Onfale") |
| Headline/H2 | Noto Serif Display SemiBold | 80px | 1.0 | 0 | 변형명 (예: "Piccolo") |
| Subheadline/S2 | Pretendard SemiBold | 40px | 1.0 | 0 | 섹션 라벨 (예: "색상") |
| Body/B1_regular | Pretendard Regular | 40px | 1.44 | -0.8 | Hero 한글 서브카피 |
| Body/B1_medium | Pretendard Medium | 40px | 1.44 | -0.8 | 브랜드 스테이트먼트 본문 |
| Paragraph/P2_medium | Pretendard Medium | 32px | 1.0 | 0 | 옵션 값 (예: "White") |

핵심 발견:
- **세리프 = Noto Serif Display** (Playfair 아님!). Noto Serif Display는 **한글도 지원** → 제품명을 한글로 써도 됨
- **본문이 40px** — 우리 기존 17px의 약 2.4배. 모바일 상세 이미지라 크게
- 영문 브랜드/제품명은 **세리프**, 한글 카피는 **Pretendard 산세리프**로 분리

## 4. ★ 가장 중요한 발견 — 쇼케이스 구조 (퍼널 아님)

전체 상세페이지(Onfale Medio, 1000×11206px)를 통째로 보니, **DUOMO 실제 구조는
판매 퍼널이 아니라 "제품 쇼케이스"**다.

**DUOMO 실제 섹션 순서 (실측):**
1. **Hero** — 풀블리드 제품 사진 + [한글 서브카피 → 제품명 세리프 → 변형명 세리프]
2. **제품 소개 문단** — 짧은 한글 설명 (제품의 형태·용도)
3. **디자이너** — 흑백 인물 사진 + 캡션 "Designer | Luciano Vistosi"
4. **브랜드 스테이트먼트** — 로고 + 따옴표 + 브랜드 철학 한 문장
5. **제품 특징 + 라이프스타일 사진** (교대 반복, 풀블리드 사진 ↔ 흰 배경 카피)
6. **색상 옵션** — 둥근 카드 사진 + 라벨 + 원형 스와치
7. **치수 도면** — 라인 드로잉 + 실측 치수 (예: ø280 × 345 × ø120)

**우리 기존 13섹션과의 결정적 차이:**

| 우리 13섹션 (서구 DTC 퍼널) | DUOMO 실제 (제품 쇼케이스) |
|---|---|
| Hero → **Pain → Problem** → Story | Hero → 제품소개 → **디자이너** |
| → Solution → How → **Social Proof** | → 브랜드철학 → 특징/라이프스타일 |
| → Authority → Benefits → **Risk Removal** | → 색상옵션 → **치수도면** |
| → **Comparison → Target Filter → Final CTA** | (판매압박 섹션 없음) |

→ DUOMO는 **고통·공포·긴급성·비교·CTA 강요가 없다.** 차분히 제품·디자이너·브랜드·
스펙을 보여준다. "30년 가는 정통"을 **말로 설득하지 않고 사진과 스펙으로 보여줌**.

**시사점**: 우리 도구의 13섹션 템플릿(원래 landing-page-generator 스킬 = 고전환
퍼널)은 DUOMO 톤과 구조적으로 안 맞는다. **새 섹션 세트 권장**:
`hero · intro · designer · brand · feature×N · lifestyle×N · color · dimensions`

## 4b. 섹션 구성 패턴 (실측 디테일)

관찰된 섹션 유형:

**A. Hero (풀블리드 제품 사진 + 텍스트 오버레이)**
- 제품 사진이 화면 전체
- 하단 중앙에 텍스트 블록: [한글 서브카피 40px] → [제품명 세리프 106px] → [변형명 세리프 80px]
- 텍스트는 흰색, 중앙 정렬, 세로로 쌓임 (gap 32px)

**B. 브랜드 스테이트먼트 (순백 배경)**
- 상단: 브랜드 로고 (벡터, 브랜드 컬러)
- 중앙: 큰 따옴표(") 장식 → 브랜드 철학 한글 3줄 (Pretendard Medium 40px, #2C2C2C) → 닫는 따옴표
- 전부 중앙 정렬, 위아래 270px 여백
- 예: "Artemide는 '빛을 통한 인간의 편안하고 자연스러운 경험'을 추구하며 전 세계 조명 디자인 분야를 선도하는 기업입니다."

**C. 옵션/스펙 (순백 배경)**
- 둥근 모서리 카드(radius ≈ 폭의 11.6%, 920px→107px)에 제품 사진
- 아래 중앙: 라벨(Pretendard SemiBold 40px "색상") → 옵션 (원형 스와치 + 값 "White" 32px)
- 색상 스와치 = 52px 원, #C4C4C4 테두리

**D. 디자이너 (흑백 인물)**
- 디자이너 흑백 포트레이트 사진 (풀블리드 또는 큰 카드)
- 캡션 "Designer | Luciano Vistosi" (이름 영문, 흰색/회색)
- 제품 = 누가 디자인했는가를 인물로 보여줌 (헤리티지 강조)

**E. 치수 도면 (스펙의 핵심)**
- 제품 라인 드로잉(실루엣)에 치수 표기: ø280 × 345 × ø120 (지름·높이)
- 중앙 정렬, 흰 배경, 회색 라인
- "정보" 백본 — 럭셔리도 결국 정확한 스펙을 보여줌

**F. 제품 소개 문단 (Hero 직후)**
- Hero 다음에 짧은 한글 문단 (제품의 형태·용도·자리)
- Pretendard, 중앙 정렬, 흰 배경
- 예: "둥근 ... Onfale은 ... 테이블 위 사이드 ... 그立체적 ... 형태의 공간..."

**G. 라이프스타일 (풀블리드 인테리어)**
- 제품이 실제 공간(거실·콘솔·식탁)에 놓인 사진, 풀블리드
- 캡션 최소 또는 없음. 사진이 말하게
- 제품 단독컷 ↔ 라이프스타일컷 ↔ 흰 카피 섹션을 교대 배치 (리듬)

채널 변형: 같은 콘텐츠를 1000px(29CM·SSG·LF·S.I VILLAGE·롯데홈쇼핑·크림 등)과
750px 두 폭으로 제작. 사이즈통합본(Grande/Medio/Piccolo 한 페이지)도 존재.

## 6. 적용 체크리스트

- [ ] 캔버스 1000px 폭으로
- [ ] 모든 텍스트 중앙 정렬
- [ ] 배경 순백, 본문 #2C2C2C
- [ ] 본문 40px / 라벨 40px SemiBold / 제품명 세리프 106px
- [ ] 세리프는 Noto Serif Display (한글 지원)
- [ ] 액센트는 해당 브랜드 고유색 (브리프에 brand_color 추가 필요)
- [ ] 섹션 상하 여백 크게 (목표 270px, 짧은 섹션은 비례 축소)
- [ ] 브랜드 스테이트먼트엔 따옴표 장식 + 철학 문장
- [ ] 옵션 섹션은 라벨 + 원형 스와치 패턴
